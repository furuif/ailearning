# 实时动态溢价率 - 进阶公式与实战模型

## 一、为什么基础公式不够用？

### 基础公式的局限
```
基础溢价率 = (P_t - P_0) / P_0 × 100%
```

**问题**：
1. ❌ 只考虑价格，忽略时间价值
2. ❌ 没有考虑市场整体波动
3. ❌ 无法区分合理溢价 vs 情绪溢价
4. ❌ 缺少风险调整
5. ❌ 难以横向比较（不同行业、不同市值）

## 二、实时动态溢价率 - 进阶公式

### 公式 1：时间衰减调整溢价率 ⭐⭐⭐⭐⭐

**核心思想**：新股溢价随时间自然衰减，需要调整

```python
def time_adjusted_premium(current_price, issue_price, days_since_ipo):
    """
    时间衰减调整溢价率
    
    逻辑：
    - 前 5 日：情绪驱动，溢价合理
    - 5-20 日：快速衰减期
    - 20-60 日：缓慢衰减期
    - 60 日后：视为正常股票
    """
    base_premium = (current_price - issue_price) / issue_price
    
    # 时间衰减因子（指数衰减）
    if days_since_ipo <= 5:
        decay_factor = 1.0  # 前 5 日不衰减
    elif days_since_ipo <= 20:
        decay_factor = np.exp(-0.05 * (days_since_ipo - 5))  # 快速衰减
    elif days_since_ipo <= 60:
        decay_factor = np.exp(-0.05 * 15) * np.exp(-0.02 * (days_since_ipo - 20))  # 慢速衰减
    else:
        decay_factor = 0.3  # 60 日后稳定在 30%
    
    adjusted_premium = base_premium / decay_factor
    
    return adjusted_premium, base_premium, decay_factor

# 示例
# 发行价 10 元，30 日后价格 15 元
# 基础溢价率 = 50%
# 调整后溢价率 = 50% / 0.45 = 111%（说明仍显著高估）
```

**适用场景**：判断新股在任意时点的相对估值水平

### 公式 2：市场调整溢价率 (Market-Adjusted Premium) ⭐⭐⭐⭐⭐

**核心思想**：剥离大盘波动影响，只看个股相对表现

```python
def market_adjusted_premium(stock_return, market_return, period='since_ipo'):
    """
    市场调整溢价率 = 个股收益 - 市场收益
    
    逻辑：
    - 如果大盘涨了 20%，新股涨了 50%
    - 真实超额收益 = 50% - 20% = 30%
    """
    excess_return = stock_return - market_return
    
    # 转换为溢价率形式
    adjusted_premium = (1 + stock_return) / (1 + market_return) - 1
    
    return adjusted_premium, excess_return

# 实战计算
def calculate_map_for_ipo(current_price, issue_price, index_start, index_current):
    """
    计算市场调整溢价率
    
    current_price: 新股当前价
    issue_price: 发行价
    index_start: 上市日大盘指数
    index_current: 当前大盘指数
    """
    stock_return = (current_price - issue_price) / issue_price
    market_return = (index_current - index_start) / index_start
    
    map_rate = (1 + stock_return) / (1 + market_return) - 1
    
    return map_rate

# 示例
# 新股：10 元→15 元，收益 50%
# 大盘：3000 点→3300 点，收益 10%
# MAP = (1.5/1.1) - 1 = 36.4%
# 解读：剔除大盘影响后，真实溢价 36.4%
```

**适用场景**：牛市/熊市中客观评估新股表现

### 公式 3：行业相对溢价率 (Industry-Relative Premium) ⭐⭐⭐⭐⭐

**核心思想**：与同行业已上市公司比较

```python
def industry_relative_premium(ipo_pe, industry_pe, ipo_growth, industry_growth):
    """
    行业相对溢价率
    
    考虑估值和成长性的相对位置
    """
```

### 公式 4：情绪调整溢价率 (Sentiment-Adjusted Premium) ⭐⭐⭐⭐

**核心思想**：分离情绪溢价和基本面溢价

```python
def sentiment_adjusted_premium(total_premium, sentiment_score, fundamentals_score):
    """
    情绪调整溢价率
    
    总溢价率 = 基本面溢价 + 情绪溢价
    
    参数:
    sentiment_score: 情绪得分 (0-100)
    fundamentals_score: 基本面得分 (0-100)
    """
    # 权重（可根据历史数据回归优化）
    sentiment_weight = 0.4
    fundamental_weight = 0.6
    
    # 分解溢价
    sentiment_premium = total_premium * sentiment_weight * (sentiment_score / 50)
    fundamental_premium = total_premium * fundamental_weight * (fundamentals_score / 50)
    
    # 情绪调整后的"真实"溢价
    adjusted_premium = fundamental_premium
    
    # 情绪泡沫比例
    bubble_ratio = sentiment_premium / total_premium
    
    return {
        'adjusted_premium': adjusted_premium,
        'sentiment_premium': sentiment_premium,
        'bubble_ratio': bubble_ratio
    }

# 情绪指标构建
def build_sentiment_index(turnover_rate, price_limit_hits, retail_ratio):
    """
    构建情绪指数
    
    turnover_rate: 换手率 (%)
    price_limit_hits: 涨停次数
    retail_ratio: 散户交易占比 (%)
    """
    # 标准化各指标
    z_turnover = (turnover_rate - turnover_rate.mean()) / turnover_rate.std()
    z_hits = price_limit_hits / 5  # 归一化
    z_retail = (retail_ratio - 30) / 20  # 假设平均 30%
    
    # 加权合成
    sentiment = 0.4 * z_turnover + 0.3 * z_hits + 0.3 * z_retail
    
    # 转换为 0-100 分
    sentiment_score = (sentiment - sentiment.min()) / (sentiment.max() - sentiment.min()) * 100
    
    return sentiment_score
```

**适用场景**：识别情绪驱动的高溢价，规避回调风险

### 公式 5：流动性调整溢价率 (Liquidity-Adjusted Premium) ⭐⭐⭐⭐

**核心思想**：考虑流动性溢价，小盘股/低流通天然高溢价

```python
def liquidity_adjusted_premium(premium, turnover_rate, float_market_cap, avg_turnover):
    """
    流动性调整溢价率
    
    逻辑：
    - 高换手→流动性好→溢价合理
    - 低换手→流动性差→溢价应打折
    - 小流通盘→稀缺性→溢价合理
    """
    # 流动性因子
    liquidity_factor = turnover_rate / avg_turnover
    
    # 稀缺性因子（小盘股溢价）
    scarcity_factor = np.log(100 / float_market_cap) / np.log(100)  # 假设基准 100 亿
    
    # 调整系数
    adjustment = 0.5 * liquidity_factor + 0.5 * scarcity_factor
    
    # 调整后溢价率
    adjusted_premium = premium / adjustment
    
    return adjusted_premium, adjustment

# 解读
# adjusted_premium > premium: 流动性折价，实际更贵
# adjusted_premium < premium: 流动性溢价，实际更便宜
```

**适用场景**：比较不同流通盘大小新股的溢价率

### 公式 6：风险调整溢价率 (Risk-Adjusted Premium) ⭐⭐⭐⭐⭐

**核心思想**：考虑波动率风险，高波动应对应高溢价补偿

```python
def risk_adjusted_premium(premium, volatility, risk_free_rate=0.03, holding_period=20):
    """
    风险调整溢价率 (夏普比率思路)
    
    premium: 持有期溢价率
    volatility: 日波动率 (%)
    risk_free_rate: 无风险利率（年化）
    holding_period: 持有天数
    """
    # 年化溢价率
    annualized_premium = (1 + premium) ** (252 / holding_period) - 1
    
    # 年化波动率
    annualized_vol = volatility * np.sqrt(252)
    
    # 风险调整溢价率（类似夏普比率）
    rap = (annualized_premium - risk_free_rate) / annualized_vol
    
    return {
        'rap': rap,
        'annualized_premium': annualized_premium,
        'annualized_vol': annualized_vol
    }

# 解读
# RAP > 1: 风险补偿充足，值得持有
# RAP 0.5-1: 风险补偿一般
# RAP < 0.5: 风险补偿不足，考虑卖出
```

**适用场景**：比较不同波动率新股的投资价值

### 公式 7：综合动态溢价率指数 (Composite Dynamic Premium Index) ⭐⭐⭐⭐⭐

**核心思想**：多因子加权，最全面的评估框架

```python
def composite_premium_index(
    base_premium,
    days_since_ipo,
    market_return,
    industry_pe_ratio,
    turnover_rate,
    volatility,
    sentiment_score
):
    """
    综合动态溢价率指数
    
    整合 7 个维度:
    1. 基础溢价率
    2. 时间衰减
    3. 市场调整
    4. 行业相对估值
    5. 流动性
    6. 风险调整
    7. 情绪调整
    """
    # 1. 时间衰减调整
    time_factor = np.exp(-0.03 * min(days_since_ipo, 60))
    time_adjusted = base_premium / time_factor
    
    # 2. 市场调整
    market_adjusted = (1 + base_premium) / (1 + market_return) - 1
    
    # 3. 行业相对调整
    industry_adjusted = base_premium / industry_pe_ratio
    
    # 4. 流动性调整
    liquidity_adjusted = base_premium / (turnover_rate / 5)  # 假设平均换手 5%
    
    # 5. 风险调整
    risk_adjusted = base_premium / volatility
    
    # 6. 情绪调整
    sentiment_adjusted = base_premium * (1 - sentiment_score / 200)  # 情绪越高，打折越多
    
    # 加权合成（权重可通过历史回归优化）
    weights = {
        'time': 0.20,
        'market': 0.20,
        'industry': 0.20,
        'liquidity': 0.15,
        'risk': 0.15,
        'sentiment': 0.10
    }
    
    cdpi = (
        weights['time'] * time_adjusted +
        weights['market'] * market_adjusted +
        weights['industry'] * industry_adjusted +
        weights['liquidity'] * liquidity_adjusted +
        weights['risk'] * risk_adjusted +
        weights['sentiment'] * sentiment_adjusted
    )
    
    # 标准化为 0-100 分
    cdpi_score = np.clip(cdpi * 100, 0, 100)
    
    # 解读
    if cdpi_score > 80:
        signal = "严重高估，强烈卖出"
    elif cdpi_score > 60:
        signal = "高估，卖出"
    elif cdpi_score > 40:
        signal = "合理，持有"
    elif cdpi_score > 20:
        signal = "低估，买入"
    else:
        signal = "严重低估，强烈买入"
    
    return {
        'cdpi_score': cdpi_score,
        'signal': signal,
        'components': {
            'time_adjusted': time_adjusted,
            'market_adjusted': market_adjusted,
            'industry_adjusted': industry_adjusted,
            'liquidity_adjusted': liquidity_adjusted,
            'risk_adjusted': risk_adjusted,
            'sentiment_adjusted': sentiment_adjusted
        }
    }
```

**适用场景**：综合评估新股投资价值，量化交易决策

## 七、关键公式速查表

```
┌─────────────────────────────────────────────────────────┐
│ 基础公式                                                 │
│ 溢价率 = (市价 - 发行价) / 发行价 × 100%                │

│ 时间调整                                                 │
│ 调整后 = 基础溢价 / exp(-0.03 × 天数)                   │
```
